# 巡检宝部署运行说明

## 本地开发运行

```bash
pnpm install
pnpm db:generate
pnpm db:push
pnpm db:seed
pnpm dev:api
pnpm dev:admin
```

- 后端默认地址：`http://127.0.0.1:3010/api/v1`
- 前端默认地址：`http://127.0.0.1:5180/`
- 前端接口地址可通过 `VITE_API_BASE_URL` 配置。

## Docker 试部署

```bash
docker compose up --build -d
```

- 管理端入口：`http://服务器IP:8080/`
- 后端接口：`http://服务器IP:3010/api/v1`
- 管理端容器内会把 `/api/v1` 代理到后端服务。

## 数据与文件

- SQLite 数据库存放在 `api-data` volume。
- 地图和报告上传文件存放在 `api-storage` volume。
- 本地开发数据库 `services/api/dev.db` 不进入 Git。
- 本地上传目录 `services/api/storage` 不进入 Git。

## 首次初始化

Docker 后端启动时会自动执行：

```bash
pnpm db:push
```

这会同步数据库表结构，但不会自动清空数据。演示环境如果需要初始化示例数据，可进入容器执行：

```bash
docker compose exec api pnpm db:seed
```

注意：`db:seed` 会重置开发示例数据，正式客户数据环境不要随意执行。

## 公有云试部署建议

- 云服务器建议先使用 2 核 4G 以上配置。
- 开放端口：`8080`、`3010`。正式上线建议只开放 `80/443`，由 Nginx 或云负载均衡转发。
- 上传文件较多后，应把 `api-storage` 替换为对象存储。
- 正式生产数据库建议从 SQLite 切换为 PostgreSQL 或云数据库。
- 域名 HTTPS、登录权限、操作审计应在试部署验证后进入正式上线模块。

## 当前限制

- 当前 Docker 配置用于试部署和演示，不是最终生产安全配置。
- 后端仍使用 SQLite 作为演示数据库。
- 前端静态资源由 Nginx 提供，API 通过 Nginx 反向代理到后端。
