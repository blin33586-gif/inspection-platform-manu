# 巡检宝平台产品方案

## 1. 项目定位

巡检宝是面向街道、城管、社区治理场景的无人机巡检档案与问题管理平台。第一阶段重点服务上海市虹口区曲阳路街道，将无人机巡检过程中形成的报告、照片、视频、问题点位和整改记录，沉淀为可查询、可汇总、可展示的城市管理档案。

平台不是单纯的文件存储系统，而是以“街道城市管理对象”为主线组织数据：

- 小区：飞线整治、违建、装修垃圾、绿化、消防通道、楼顶杂物等。
- 道路街面：占道经营、流动摊贩、门头广告牌、非机动车乱停、沿街堆物等。
- 重点点位：反复发生问题的位置、重点广告牌、河道岸线、绿化带、建筑工地、垃圾堆放点等。
- 巡检报告：按日期、区域、类型形成可查看、可下载、可归档的报告资料。
- 问题台账：记录发现、整改、复查、归档的全过程。

## 2. 服务对象

### 2.1 街道管理人员

主要诉求：

- 快速查看本街道近期巡检情况。
- 按小区、道路、问题类型查找历史资料。
- 向上级或内部会议展示无人机巡检成果。
- 追踪重点问题是否整改。

### 2.2 城管工作人员

主要诉求：

- 查看具体问题的照片、位置和报告。
- 按类型筛选问题，例如飞线、违建、占道经营、广告牌。
- 查看某个点位是否反复出现问题。
- 作为执法、约谈、整治、复查的辅助材料。

### 2.3 项目运营人员

主要诉求：

- 上传巡检报告、照片、视频等资料。
- 整理每次巡检发现的问题。
- 维护小区、道路、重点点位档案。
- 生成阶段性服务成果展示。

## 3. 第一版建设目标

第一版目标是完成“可展示、可查询、可积累”的基础平台。

### 3.1 必须实现

- 客户可以登录平台。
- 首页可以看到街道整体巡检数据。
- 可以按小区查看巡检报告、照片和问题。
- 可以按道路查看巡检报告、照片和问题。
- 可以管理重点点位。
- 可以上传 PDF 报告、现场照片、短视频。
- 可以建立问题台账。
- 可以按状态查看问题：待处理、已整改、复查通过、暂不处理。
- 可以通过手机端或小程序入口查看资料。

### 3.2 第一版暂缓

- AI 自动识别问题。
- 无人机实时直播。
- 三维地图。
- 复杂派单流转。
- 多部门复杂权限。
- 自动生成执法文书。

这些能力可以在第二阶段或第三阶段逐步加入。

## 4. 业务模块

### 4.1 首页驾驶舱

首页用于让客户快速理解当前街道的管理态势。

核心内容：

- 本月巡检次数。
- 本月发现问题总数。
- 已整改数量。
- 未整改数量。
- 复查中数量。
- 问题类型分布。
- 高频问题小区排行。
- 高频问题道路排行。
- 最新巡检报告。
- 重点问题地图。

建议首页采用“工作台 + 地图 + 统计”的形式，不做过度夸张的大屏风格。

#### 4.1.1 首页二维地图

首页地图支持使用街道二维底图，底图来源可以是：

- TIF / TIFF 格式地图。
- JPG / PNG / WebP 格式图片地图。
- 后续接入高德、腾讯等在线地图。

客户希望的交互方式是可行的：首页加载曲阳路街道总览图，在地图上显示小区名称、道路名称和重点点位。用户点击“小区名称”或“小区范围”，进入对应小区档案；点击“道路名称”或道路热区，进入道路档案；点击重点点位，进入点位详情。

地图呈现建议：

- 小区用半透明色块或边界线圈出，不同小区可以使用不同颜色。
- 小区名称直接显示在对应范围中心位置。
- 道路用线条或长条热区标识，显示道路名称。
- 问题点位用状态颜色标识，例如红色待处理、橙色处理中、绿色已整改。
- 支持缩放、拖拽、图层开关和搜索定位。

TIF / TIFF 文件不建议直接作为网页图片加载。更稳妥的方式是先把 TIF 地图处理成网页适合加载的格式：

- 如果只是普通图片地图：转换为 PNG / WebP，再在前端加载。
- 如果文件很大：切成地图瓦片，再按缩放级别加载。
- 如果 TIF 自带地理坐标：保留坐标信息，转换成可用于网页地图的瓦片或 GeoJSON 配套数据。
- 如果小区已经单独拆分：每个小区可以作为独立图层或独立热区绑定到小区档案。

第一版推荐做法：

1. 后台上传一张街道总览底图，格式支持 PNG / JPG / WebP，TIF 由后台转换。
2. 在后台维护小区、道路、重点点位的可点击范围。
3. 首页地图显示小区名称、道路名称和问题点位。
4. 点击地图对象进入对应档案。
5. 后续再升级为标准地理信息地图或瓦片地图。

### 4.2 小区档案

以小区作为一个长期管理单位。

每个小区档案包含：

- 小区名称。
- 所属街道。
- 所属居委。
- 地址。
- 联系人或责任人。
- 小区边界或中心点位置。
- 小区标签，例如老旧小区、飞线高发、违建重点、建筑垃圾高发。
- 历史巡检记录。
- 历史问题记录。
- 关联报告。
- 关联照片和视频。

常见问题类型：

- 飞线充电。
- 楼顶违建。
- 违规搭建。
- 装修垃圾未清运。
- 公共区域堆物。
- 绿化破坏。
- 消防通道占用。
- 道路破损或积水。

### 4.3 道路街面档案

以道路为管理单位，适合街面秩序和沿街商铺管理。

每条道路档案包含：

- 道路名称。
- 起止范围。
- 所属片区。
- 重点商铺或门头点位。
- 历史巡检记录。
- 历史问题记录。
- 关联报告。
- 关联照片和视频。

常见问题类型：

- 占道经营。
- 流动摊贩。
- 店外经营。
- 门头广告牌异常。
- 违规横幅。
- 非机动车乱停。
- 沿街堆物。
- 垃圾桶外溢。

### 4.4 重点点位

重点点位用于长期追踪反复出现问题的位置。

点位类型：

- 反复占道点位。
- 重点广告牌。
- 河道岸线。
- 绿化带。
- 建筑工地。
- 垃圾堆放点。
- 屋顶违建点。
- 小区出入口。

每个点位需要有时间轴，记录每次巡检拍到的照片和状态变化。

### 4.5 巡检报告中心

报告中心是客户最容易理解和接受的入口。

支持筛选：

- 巡检日期。
- 小区。
- 道路。
- 重点点位。
- 问题类型。
- 报告状态。

报告内容：

- PDF 报告。
- 报告封面图。
- 巡检区域。
- 巡检时间。
- 发现问题数量。
- 关联问题列表。
- 关联照片。
- 关联视频。

### 4.6 问题台账

问题台账是平台后续升级的核心。

每条问题记录包含：

- 问题编号。
- 问题标题。
- 所属对象：小区、道路或点位。
- 问题类型。
- 发现时间。
- 发现方式：无人机巡检、人工补录、客户反馈。
- 问题描述。
- 详细位置。
- 经纬度。
- 严重程度：一般、较重、严重。
- 当前状态：待处理、处理中、已整改、复查通过、暂不处理。
- 巡检照片。
- 复查照片。
- 关联报告。
- 备注。

第一版可以先做“展示型台账”，后续再升级为派单闭环。

### 4.7 文件资料库

用于统一管理图片、视频、PDF 等资料。

文件分类：

- 巡检报告。
- 原始照片。
- 标注照片。
- 视频片段。
- 整改前后对比图。
- 其他附件。

存储建议：

- 文件放对象存储，例如阿里云 OSS、腾讯云 COS、华为云 OBS。
- 数据库只保存文件地址、文件类型、关联业务对象和上传时间。
- 大文件不要直接放服务器本地磁盘。

## 8. 技术方案建议

### 8.1 推荐架构

第一版建议采用常规 Web 管理后台 + 服务端 API + 对象存储 + 小程序查看端。

建议组合：

- 前端管理后台：Vue 3 或 React。
- 后端服务：Node.js / NestJS、Java Spring Boot 或 Python FastAPI。
- 数据库：PostgreSQL 或 MySQL。
- 文件存储：阿里云 OSS、腾讯云 COS 或华为云 OBS。
- 地图：高德地图或腾讯地图。
- 部署：阿里云 ECS / 腾讯云轻量服务器 + 对象存储 + 云数据库。

如果后续重点做政企项目，建议第一版就把部署、安全和数据备份考虑进去。

### 8.1.1 TIF 地图加载技术路线

TIF / TIFF 在浏览器里的兼容性和加载性能不适合直接面向客户使用，尤其是无人机正射图、街道高清图这类文件通常较大。平台应当把 TIF 当作原始地图资产，上传后由后台处理成前端可展示的地图。

可选路线：

- 普通图片路线：TIF 转 PNG / WebP，适合小范围、小文件、第一版快速上线。
- 图片热区路线：在转换后的图片上绘制小区、道路、点位热区，点击热区跳转到档案。
- 瓦片地图路线：把大 TIF 切成瓦片，前端按缩放级别加载，适合高清大图。
- 地理坐标路线：如果 TIF 是 GeoTIFF，可以保留地理坐标，结合小区边界、道路线、问题点位做标准 GIS 展示。

第一版建议先采用“图片地图 + 热区”的路线，开发成本低，客户也容易理解。等曲阳路街道样板跑通后，再升级为“瓦片地图 + 地理坐标”。

### 8.2 推荐第一版技术选择

为了开发速度和后续维护，建议：

- Web 前端：React + Ant Design。
- 后端：Node.js + NestJS。
- 数据库：PostgreSQL。
- 文件：阿里云 OSS。
- 小程序：微信小程序原生或 Taro。

理由：

- 后台管理界面开发快。
- 表格、筛选、上传、弹窗等功能成熟。
- 后续扩展统计、地图、权限比较方便。
- 小程序可以复用接口。

## 10. 后续升级方向

### 10.1 巡检闭环

从“看资料”升级为“发现问题、派发整改、上传结果、复查归档”。

### 10.2 AI 辅助识别

对飞线、违建、占道经营、垃圾堆放等高频问题做自动识别或半自动标注。

### 10.3 自动生成报告

根据巡检记录和问题台账自动生成 PDF 报告，减少人工排版。

### 10.4 多街道复制

将曲阳路街道作为样板，后续复制到其他街道或区级平台。

### 10.5 数据分析

分析高发区域、高发问题、整改效率、反复问题，为街道治理提供决策依据。
