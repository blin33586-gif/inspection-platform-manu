## 2026-06-27 文件上传与地图资产处理

### 本阶段目标

- 支持地图文件上传入口。
- 支持常见图片格式和 `.tif` / `.tiff` 地图文件。
- 上传后在地图资产表生成记录，保留原文件名、存储文件名、文件大小、MIME 类型、来源类型和处理状态。
- 文件本体保存到后端本地存储目录，后续部署时可替换为对象存储。

### 新增能力

- `POST /api/v1/map-assets/upload`
- 表单字段：
  - `file`：地图文件。
  - `name`：地图名称，可选。
  - `mapType`：地图类型，可选。
- 本地存储目录：`services/api/storage/map-assets`。

### 已验证

- `CI=true pnpm --filter @xunjianbao/api db:generate`
- `CI=true pnpm --filter @xunjianbao/api db:push`
- `CI=true pnpm --filter @xunjianbao/api db:seed`
- `CI=true pnpm build`
- 使用临时 `.tif` 文件验证上传接口，返回 `sourceType: "tiff"`。
- `GET /api/v1/map-assets` 可读取新上传地图资产。

### 冻结说明

- 本阶段只完成上传、识别和入库。
- TIFF 切片、坐标标定、热区绘制、图片预览作为后续地图深化模块继续开发。
- 本地上传文件目录不进入 Git。
